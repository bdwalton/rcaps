Class Caps
In: rcaps.c
Parent: Object

RCaps - Ruby Capabilities

Manipulate Linux Kernel/Process capabilities from Ruby.

For information about how capabilities work, see: man 7 capabilities

Author:Ben Walton bdwalton@gmail.com

Examples

Fetch running capabilities and modify them.

require ‘rcaps‘

c = Caps.get_proc

c.set_inheritable([Caps::SETUID])

c.set_proc

Start with capabilities in string form and modify them.

require ‘rcaps‘

s = ’= cap_chown,cap_setuid+e‘

c = Caps.new(s)

c.set_permitted([Caps::MKNOD])

c.set_proc

Start with an empty capability set and modify it.

require ‘rcaps‘

c = Caps.new

c.set_permitted([Caps::MKNOD])

c.set_proc

Fetch running capabilities and clear them.

require ‘rcaps‘

c = Caps.get_proc

c.clear

c.set_proc

Fetch running capabilities from another process and clear them.

require ‘rcaps‘

c = Caps.get_proc(1234)

c.clear

c.set_proc(1234)

Fetch running capabilities and ensure a certain capability is not set.

require ‘rcaps‘

c = Caps.get_proc

c.clear_effective([Caps::SETUID])

c.clear_permitted([Caps::SETUID])

c.clear_inheritable([Caps::SETUID])

c.set_proc

Strip current capabilities.

require ‘rcaps‘

c = Caps.new

c.set_proc

Fetch running capabilities and see if a certain privilege is granted.

require ‘rcaps‘

c = Caps.get_proc

puts "I can change my UID‘s!" if c.effective?(Caps::SETUID)

Public Class methods

get_proc(...)

Returns a new Caps object initialized with the set of capabilities from the currently running process. If passed a pid, the capabilities for that process will be retrieved instead.

new(...)

Returns an object representing a capability set in working memory. If an argument is passed, it must be a string representing a set of capabilities in string form. This is analagous to obtaining a capability set by calling the C level routine cap_from_text.

Public Instance methods

clear()

Removes all capabilties from the Caps object. This is analogous to the C level function cap_clear.

clear_effective(p1)

Clear the Capabilities listed in the first argument array from the effective set.

p1 must be an array containing Capabilities defined as constants in the Caps class.

clear_inheritable(p1)

Clear the Capabilities listed in the first argument array from the inheritable set.

p1 must be an array containing Capabilities defined as constants in the Caps class.

clear_permitted(p1)

Clear the Capabilities listed in the first argument array from the permitted set.

p1 must be an array containing Capabilities defined as constants in the Caps class.

effective?(p1)

Return a boolean response indicating whether a capability is effective or not within the working set.

p1 is a constant referring to a Capability as defined in the Caps class.

inheritable?(p1)

Return a boolean response indicating whether a capability is inheritable or not within the working set.

p1 is a constant referring to a Capability as defined in the Caps class.

permitted?(p1)

Return a boolean response indicating whether a capability is permitted or not within the working set.

p1 is a constant referring to a Capability as defined in the Caps class.

set_effective(p1)

Add the Capabilities listed in the first argument array to the effective set.

p1 must be an array containing Capabilities defined as constants in the Caps class.

set_inheritable(p1)

Add the Capabilities listed in the first argument array to the inheritable set.

p1 must be an array containing Capabilities defined as constants in the Caps class.

set_permitted(p1)

Add the Capabilities listed in the first argument array to the permitted set.

p1 must be an array containing Capabilities defined as constants in the Caps class.

set_proc(...)

Install the Caps object into the kernel. This is analogous to the C level function cap_set_proc. If a pid is given, the capabilities will be installed for that process instead. [Note: This likely won‘t work, but we support it anyway]

to_s()

Returns a String object that contains the text representation of the Caps object. This is analagous to the C level function cap_to_text.
